2D Drawing：Thin Line 与 Circle

Revision 明确列出的 Object Modeling 起点：把连续几何转换为 frame buffer 中的像素模式

⚡ 基础算法步骤
📌 讲义原话口径：Drawing an object means converting the object into a pixel pattern and updating the corresponding pixels in the frame buffer. Thin line 和 circle 的共同目标都是选择最接近真实几何的像素。

Thin Line

直线由两点定义，基础方程是 y = mx + c。讲义先讨论 0 ≤ m ≤ 1 的情况：每次令 x 增加 1，再比较真实直线位置到上方/下方像素中心的距离，选择更近的像素。

Bresenham 核心优势是只用整数运算；其他斜率可通过交换 x/y 或改变符号利用对称性处理。

Circle

圆由 (x - xc)2 + (y - yc)2 = r2 定义。算法目标是在像素网格中选择使 |(x - xc)2 + (y - yc)2 - r2| 最小的像素。

Symmetry 圆具有 8 对称性，只需计算一个八分区，其余点通过对称复制；Bresenham circle 同样用整数决策变量提高效率。

对象讲义关键词考场回答重点
Thin liney = mx + c；upper/lower pixel center；Bresenham line逐步选择更接近真实直线的像素；Bresenham 用整数运算；利用对称性覆盖其他斜率
Circlecircle equation；closest pixel；symmetry；Bresenham circle用圆方程残差判断像素接近程度；利用 8 对称性减少计算
⚠️ 答题边界：Revision 只点名 a line 和 a circle。闭卷时优先写"连续几何 → pixel pattern"和"选择最接近真实几何的像素"，不需要背整段 C 代码。

多边形网格深入

数据结构、法向量定向、Euler 公式、可定向性 — M02 中最重要的考点集中地

⚡ Euler 公式是高频计算考点

数据结构：Face-Vertex 表示

顶点表 (Vertex Position)

(x₁, y₁, z₁)
(x₂, y₂, z₂)
...
(xₙ, yₙ, zₙ)

面表 (Triangle Face Index)

(f₁₁, f₁₂, f₁₃)
(f₂₁, f₂₂, f₂₃)
...
(fₙ₁, fₙ₂, fₙ₃)
📌 右手定则：顶点按逆时针 (CCW) 排列 → 法向量朝外 (outward facing)。使用右手定则判断：四指弯向顶点顺序，大拇指指向法向量方向。

Euler 公式

标准 Euler 公式（凸多面体，无洞）：
V − E + F = 2

广义 Euler 公式（亏格 g，边界数 b）：
V − E + F = 2 − 2g − b
多面体VEFV − E + F
四面体 (Tetrahedron)4642
正方体 (Cube)81262
八面体 (Octahedron)61282
⚠️ 陷阱：Euler 公式 V - E + F = 2 只对凸多面体（无洞的封闭流形）成立！有洞/把手 (亏格 g ≥ 1) 时用广义公式 V - E + F = 2 - 2g。例如：甜甜圈 (torus) 的 g = 1，V - E + F = 0。

可定向 vs 不可定向

类型定义例子
Orientable（可定向） 所有面可一致定向（全部 CCW 或全部 CW），每条边对两个相邻面方向相反 大多数实体模型（球、立方体、甜甜圈）
Non-orientable（不可定向） 无法一致定向，沿某些路径绕一圈后法向量方向翻转 Möbius 带、Klein 瓶

参数曲面 vs 多边形网格

M02 最核心的对比 — 期中 Q4 原型，期末变形题高概率出现

⚡ 最高频考点：比较 + Tessellation
对比维度参数曲面 (Parametric)多边形网格 (Polygon Mesh)
数学精度✅ 精确表示（数学方程定义，无限分辨率）❌ 近似（平面多边形逼近曲面）
渲染效率❌ 需先 Tessellation（曲面细分）✅ 直接光栅化，GPU 硬件优化
编辑性控制点操作，直觉性稍差顶点/边直接操作，灵活
适用场景工业设计、精密制造 (CAD/CAM)游戏、电影、实时渲染
存储少量控制点 + 方程全部顶点 + 面索引

Tessellation 步骤（参数曲面进入光栅化管线的必经之路）

Step 1: 参数采样 在参数域 (s, t) 上按密度采样，生成离散点
Step 2: 三角剖分 将采样点连接成三角网格（类似 Triangulation）
Step 3: 输出三角形 生成的三角网格进入标准光栅化管线
✅ 考场一句话总结：参数曲面数学精度高但必须先 Tessellation 才能进入光栅化管线；多边形网格直接兼容 GPU 光栅化但曲面用平面近似存在精度损失。
⚠️ 考试关键：参数曲面进入光栅化之前必须 Tessellation——这是二者在渲染管线中最本质的区别。多边形网格的缺点是 "curved surfaces approximated by flat polygons"，导致近看有 Faceting 失真。

Point Cloud 三性质

来自 Tutorial 的精确表述，答题时可直接引用

⚡ 简答题：为什么点云难以直接渲染

1. Unstructured

点之间没有规则的拓扑关系，不能像图像像素那样直接按固定网格处理；这解释了为什么点云通常需要先重建或转换后再进入常规图形管线。

2. Sampled from 2D manifolds

点云是从物体表面（二维流形）采样得到的。这是点云的几何本质——它描述的是一个面而非一个体。

3. No topology between points

不同点之间不存在边或面的连接关系。与多边形网格对比时，这是点云最核心的局限。

优点缺点
直接从传感器获取 (LiDAR)，无需建模 无连接性，几何计算困难
表示灵活，适合稀疏数据省内存 曲面重建困难
广泛用于自动驾驶 难以直接渲染，需先重建为 Mesh
🚫 考试边界：点云只要求掌握基本定义、无拓扑限制、与 Mesh 的对比；不要展开 Revision 排除的扩展方法。

其余表示方法速查

细分曲面、隐式曲面、Voxel、CSG、Fractals — 考试只需一句话定义

Subdivision Surface

对粗糙多边形网格反复细分（加入新顶点并调整位置），在极限状态下趋近光滑曲面。

代表算法 Catmull-Clark

优点 少量控制点表示光滑曲面

缺点 极限面难精确控制

Implicit Surface

由隐函数 f(x, y, z) = 0 定义；f 的正负可用于 inside/outside test，但符号方向取决于约定。按本课 SDF 讲义：inside 为 +d(x)，outside 为 -d(x)，表面为 f=0。

代表 x2 + y2 + z2 - r2 = 0（球面）

优点 碰撞检测/内外判断 O(1)

缺点 可视化需 Marching Cubes

Voxel

3D 空间离散为等大立方体单元（三维像素）。

场景 医学影像 (CT/MRI)

缺点 高分辨率时存储爆炸

CSG

通过布尔运算 (∪, ∩, −) 组合基本几何体（球、柱、方块）。

场景 CAD 建模

缺点 显示需转为网格

Fractals

用自相似递归规则定义的无限细节几何结构。如科赫雪花、Mandelbrot 集、程序化山地地形。

优点 自然感强（山川、海岸线）

缺点 无法精确控制最终形状

Q0. Revision 中的 2D Drawing: a line, a circle 应该按什么口径复习？

总口径：Drawing an object = converting the object into a pixel pattern and updating the corresponding pixels in the frame buffer.

Thin line：用 y = mx + c 表示线段；在 0 ≤ m ≤ 1 时逐步增加 x，比较真实直线到上/下像素中心的距离，选择更近像素。Bresenham line 的要点是用整数运算高效完成这个决策；其他斜率靠交换 x/y 或变号利用对称性处理。

Circle：用 (x - xc)2 + (y - yc)2 = r2；每步选择使圆方程残差最小的像素；利用圆的 8 对称性减少计算。

Q1. (P2-1a) 验证正八面体满足 Euler 公式 V − E + F = 2

正八面体：V = 6（上下各一个顶点，中间四个），E = 12，F = 8

V - E + F = 6 - 12 + 8 = 2 ✓

Q2. (P2-1b) 亏格 g=1 的闭合网格有 V=9000, E=27000，求面数 F

广义 Euler 公式：V - E + F = 2 - 2g

g = 1 ⇒ V - E + F = 2 - 2(1) = 0

9000 - 27000 + F = 0

F = 18000

Q5. 解释 Point Cloud 的三个关键性质，并说明为什么点云难以直接渲染

三个性质：

① Unstructured — 点之间没有规则的拓扑关系，无法像图像像素那样直接按固定网格处理

② Sampled from 2D manifolds — 点云是从物体表面（二维流形）采样得到的

③ No topology between points — 不同点之间不存在边或面的连接关系

难以渲染的原因：光栅化管线需要三角形面片（polygon faces）作为输入。点云只有离散点，没有边/面信息，无法直接输入光栅化器。需要先经过曲面重建 (Surface Reconstruction) 转为 Mesh 才能渲染。

Q7. 什么是可定向 (Orientable) 网格？Möbius 带为什么不可定向？

可定向：所有面可一致定向（全部 CCW 或全部 CW），每条边对两个相邻面方向相反。大多数实体模型为可定向。

Möbius 带不可定向的原因：Möbius 带只有一个面和一条边界。沿带子走一圈后，法向量方向会翻转（原本朝外的法向量变成了朝内）。这意味着无法为所有面一致指定 CCW 方向，因此是不可定向的。

  • 2D Drawing = 连续几何 → pixel pattern → 更新 frame buffer
  • Thin line：比较真实线到上/下像素中心距离；Bresenham 用整数运算
  • Circle：用圆方程残差选最近像素；利用 8 对称性
  • V - E + F = 2（凸多面体）
  • V - E + F = 2 - 2g - b（广义，亏格 g，边界 b）
  • 四面体 (4,6,4)、正方体 (8,12,6)、八面体 (6,12,8)
  • 参数曲面 = P(s, t)；隐式曲面 = f(x, y, z) = 0
  • CCW 顶点排列 → 法向量朝外（右手定则）
  • Face-Vertex：顶点表 + 面索引表
  • Catmull-Clark = 细分曲面代表算法

考场容易遗漏的细节

  • Euler 公式只对凸多面体成立，有洞用广义公式
  • 参数曲面必须先 Tessellation 才能光栅化
  • 2D Drawing 不背代码，背"最近像素 + 对称性/整数运算"的讲义表述
  • Point Cloud 三性质：Unstructured / 2D manifolds / No topology
  • 隐式曲面可视化需要 Marching Cubes
  • Q4 变形题套路：选 Mesh 给实时场景，选 Parametric 给精度场景
  • Revision 排除的扩展专题不进入本模块答题模板
  • Möbius 带不可定向 — 沿带子走一圈法向量翻转
