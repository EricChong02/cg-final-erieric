1

光栅化过程

Primitive → 像素覆盖集合

⚡ 理解走样的前提

光栅化（Rasterization）是渲染管线中将几何图元（三角形/线段）转换为屏幕上离散像素集合的阶段。它在 Screen Space（2D 像素坐标）中执行，位于 Clipping 之后、Hidden Surface Removal 之前。

核心操作：对每个三角形 Primitive，判断屏幕上哪些像素被其覆盖
→ 被覆盖的像素进入 Fragment Processing（片段处理）
→ 最终写入 Frame Buffer
⚠️ 关键认知：光栅化采用的是点采样（Point Sampling）——对每个像素只取中心点判断是否在图元内。这是一个二值决策（覆盖/不覆盖），不关心图元在像素内实际覆盖了多大面积。这就是走样的根本来源。
📌 光栅化的本质矛盾：连续几何体（无限精度）→ 离散像素网格（有限分辨率）。无论分辨率多高，只要采样点数量有限，理论上就无法完美表示任意高频细节。
2

空间走样（Spatial Aliasing）

点采样的直接后果：锯齿、细小物体消失、形状扭曲

⚡ 高频简答题

定义：空间走样是因图像采样频率不足（点采样）导致的视觉失真，表现为锯齿、细线消失、小物体形状失真等现象。

产生机制——点采样的三种失效模式：

① 若图元覆盖采样点 → 整个像素染色（过度表示）
② 若图元未覆盖采样点 → 整个像素不染色（完全丢失）
③ 即使图元覆盖了像素面积的 50%，只要中心点没被覆盖，像素就不染色

三种典型表现形式

  • 锯齿效应（Jaggies / Staircase Effect）：斜线和曲线边缘呈阶梯状
  • 细小物体消失：三角形比像素小时，可能无法落在任何采样点上 → 整个物体消失
  • 形状扭曲：相同的两个三角形因像素网格位置不同（sub-pixel offset）而呈现不同形状

根本原因（一句话）

光栅化采用点采样（Point Sampling）的粗糙近似，采样频率低于 Nyquist 频率时，高频细节无法正确重建。

🚫 常见错误理解："走样是光栅化固有的问题"——不准确。走样的根源是采样频率不足，而非光栅化本身。提高分辨率（增加采样点）理论上可以减少走样，这正是超采样的思路。
3

时间走样（Temporal Aliasing）

空间走样在时间维度上的传播与放大

⚡ 两类原因要区分

定义：时间走样是在动画序列中，由空间走样或时间域欠采样导致的相邻帧之间图像跳变、物体闪烁或运动方向错误的视觉伪影。

原因 1：空间走样引起（主要）

  • 移动物体在不同帧采样到不同像素位置 → 锯齿形状"跳动"
  • 细小物体在相邻帧间忽显忽隐（blinking）
  • 碎片化物体在运动中不断分裂合并

📌 讲义原话："时域走样由空间走样导致"

原因 2：时间域欠采样

  • 采样频率（帧率）低于运动频率的 2 倍
  • 车轮反转效应（Wagon Wheel Effect）：车轮高速旋转但低帧率采样 → 看起来逆向旋转
  • 这是 Nyquist 定理在时间域的直接应用
4

Nyquist 采样定理

反走样所有方法的理论基石

⚡ 理论依据必考

定义：要完整重建频率不超过 Fmax 的连续信号，采样频率 N 必须至少为 2Fmax（Nyquist 频率）。

Nyquist 条件： N ≥ 2 · Fmax

在图形学中的映射：
· 采样频率 = 图像分辨率（每单位长度的像素数）
· 信号频率 = 图像中细节/纹理的空间频率
· 若图像中存在高频细节（细线、棋盘格）→ 需要更高分辨率才能正确表示
· 不满足 Nyquist 条件 → 产生走样
📌 直觉理解：对于每隔一个像素交替黑白的棋盘格图案，至少需要每像素 2 个样本才能区分。如果每个像素只取 1 个样本 → 整个图案可能被采样成全黑或全白。
⚠️ 考试关联：超采样（Supersampling）的本质就是提高采样频率以满足 Nyquist 条件。所有反走样方法都可以从"如何逼近 Nyquist 条件"的角度来理解。
5

五种反走样方法

超采样 → 累积缓冲 → 随机采样 → Catmull → A-buffer，逐级递进

⚡ 最高频对比题
5.1 超采样（Supersampling / FSAA）

核心原理：以高于最终分辨率的虚拟分辨率渲染图像，经低通滤波后降采样到目标分辨率。

步骤：
1. 以 k×k 倍的高分辨率渲染虚拟图像（k=2 → 像素数 ×4）
2. 对高分辨率图像施加低通滤波器（平均值/高斯等）
3. 降采样（Downsample）到最终目标分辨率

实际实现中步骤 2 和 3 合并：
每个最终像素 = 对应 k×k 个子像素颜色的加权平均

✅ 优点

  • 实现简单，可由硬件直接支持（讲义提到 GeForce 类图形加速卡实现）
  • 视觉效果明显改善

❌ 缺点

  • 需要大量额外存储（k² 倍帧缓冲内存）
  • 处理时间大幅增加
  • 只能减少走样，不能消除——本质上仍是点采样，只是更密
🚫 高频陷阱："超采样能消除走样"——错误！超采样只能减少走样，不能消除。因为它本质上仍然是点采样（只是密度更高），除非采样密度达到无穷大。
5.2 累积缓冲（Accumulation Buffer）⚠️ 额外优势考点

核心原理：通过多次渲染将图像在缓冲区中累加平均，用时间换空间，解决超采样内存消耗过大的问题。

步骤：
设最终分辨率 m×n，子像素分辨率 a×b

for i in 1..a×b:
  以子像素位置偏移渲染一帧（分辨率 m×n，而非 k·m × k·n）
  将结果累加到 Accumulation Buffer

最终像素颜色 = Accumulation Buffer / (a×b)

✅ 优点

  • 只需一个额外的正常分辨率缓冲区（比超采样省内存）
  • 可实现运动模糊（Motion Blur）：沿时间轴累加多帧
  • 可实现景深（Depth of Field）：改变相机位置累加

❌ 缺点

  • 需要多次完整渲染（a×b 次），时间开销大
  • 累积缓冲需要更高位深（防止多次累加溢出）
⚠️ 超级考点：Accumulation Buffer 相对普通 Supersampling 的额外优势

运动模糊（Motion Blur）和景深（Depth of Field）是 Accumulation Buffer 的额外能力。

原因：普通 supersampling 只是在一个瞬间对同一帧的多个子像素位置采样；而 Accumulation Buffer 可以跨时间（多帧）或跨相机位置（多视角）累加，这是本质区别。

📌 考场直接写："Accumulation Buffer 区别于 Supersampling 的关键优势是可以实现 Motion Blur 和 Depth of Field。"
5.3 随机采样（Stochastic Sampling）

核心原理：用非均匀随机样本点代替规则网格样本点，将走样（有规律的锯齿伪影）转换为随机噪声，从视觉上更容易接受。

常用分布：
① Poisson-Disc：设定最小间距约束，保证样本均匀分布（最常用）
② Jittered（抖动）：在规则网格基础上每个样本加随机扰动
③ 完全随机：均匀随机，可能产生聚集（不推荐）

✅ 优点

  • 相同样本数下比超采样更好地减少走样（规律锯齿 → 随机噪声）
  • 理论上样本足够时可消除走样
  • 人眼对随机噪声的容忍度远高于规律性锯齿

❌ 缺点

  • 引入随机噪声（在低样本数时尤为明显）
  • 仍需大量存储和计算
📌 核心洞察：Stochastic Sampling 的关键区别在于——规则采样 → 走样是有规律的伪影（人眼非常敏感）；随机采样 → 走样变为随机噪声（人眼容忍度高）。相同视觉质量下所需样本数更少。
5.4 Catmull 算法

核心原理：将每个多边形精确裁剪到每个像素区域，计算实际覆盖面积并加权颜色，实现最精确的反走样（精确面积加权，而非点采样）。

步骤：
对每个像素：
1. 将所有多边形裁剪到该像素的矩形区域 → 形成多边形碎片
2. 确定深度最近的可见碎片
3. 计算每个碎片在像素内的面积
4. 面积 × 碎片颜色，加权求和
5. 结果为该像素最终颜色（精确的覆盖率加权平均）

✅ 优点

  • 精确计算每个多边形对像素的颜色贡献（覆盖面积权重）
  • 理论最优的反走样质量

❌ 缺点

  • 多边形裁剪和深度排序计算量极大
  • 实际应用中效率过低，不常用于实时渲染
5.5 A-buffer 方法

核心原理：每个像素维护一个可见碎片链表（而非单一深度值），利用子像素掩码（subpixel mask）高效计算覆盖率，是 Catmull 的实用化改进。

每像素存储：
· 一个链表，包含所有覆盖该像素的多边形碎片
· 每个碎片有：深度值、颜色、子像素覆盖掩码（bitmask，如 4×4=16 bits）

计算像素颜色：
用位运算（bitwise AND/OR）高效计算子像素掩码的覆盖面积 → 加权各碎片颜色

✅ 优点

  • 处理代价只与可见碎片数有关，不与总分辨率成正比
  • 可用位运算高效处理子像素掩码
  • 可处理透明片段（链表结构天然支持）

❌ 缺点

  • 本质上仍是超采样算法（子像素掩码 = 固定位置的子采样点）
  • 链表内存管理在硬件中实现复杂

考场答案模板

题型 1：解释空间走样的产生原因与表现（P6-1 类）

答案框架：

① 定义：空间走样是因图像采样频率不足导致的视觉失真。

② 根本原因：光栅化采用 点采样（Point Sampling），对每个像素只取中心点判断图元是否覆盖——覆盖则全染，不覆盖则全不染。这违反了 Nyquist 定理。

③ 三种表现：(a) 锯齿效应 Jaggies——斜线呈阶梯状；(b) 细小物体消失——三角形小于像素时无法落在采样点上；(c) 形状扭曲——相同三角形因 sub-pixel offset 呈现不同形状。

题型 2：比较五种反走样方法（P6-2 类）

答案框架（五个维度各一句话）：

① Supersampling：高分辨率渲染 → 低通滤波 → 降采样。简单但耗内存（k² 倍），只能减少不能消除走样。

② Accumulation Buffer：多次偏移渲染 → 累加平均。省内存但多次完整渲染，可实现 Motion Blur 和 Depth of Field。

③ Stochastic Sampling：随机非均匀采样点代替规则网格。将锯齿转化为噪声（人眼容忍度高），相同样本数下效果更好。

④ Catmull's Algorithm：裁剪多边形到每个像素，计算精确覆盖面积加权。最精确但计算量极大。

⑤ A-buffer：每像素链表 + 子像素掩码，用位运算计算覆盖率。代价与可见片段数有关，可处理透明。

题型 3：Accumulation Buffer 相比 Supersampling 的优势

标准答案（两句话）：

① 内存优势：Accumulation Buffer 只需一个正常分辨率的额外缓冲区，而 Supersampling 需要 k² 倍帧缓冲。

② 额外能力：Accumulation Buffer 可以实现运动模糊（Motion Blur，沿时间轴累加）和景深（Depth of Field，改变相机位置累加）；普通 Supersampling 只在一个瞬间对单帧采样，不提供这两个效果。

题型 4：Nyquist 定理在图形学中的含义

答案框架：

① Nyquist 定理：采样频率 N ≥ 2 × 信号最高频率 Fmax，才能完整重建连续信号。

② 在图形学中：采样频率 = 图像分辨率，信号频率 = 图像中的高频细节（细线、棋盘格等）。

③ 当场景中存在高频细节，而分辨率（采样频率）不够时 → 不满足 Nyquist 条件 → 产生走样。

④ 反走样的本质：提高采样密度以满足 Nyquist 条件（超采样）；或用更好的采样策略（随机采样）。

计算代价高（一次性渲染 k² 倍像素）很高（需要 a×b 次完整渲染）
特殊优势实现简单，硬件直接支持可实现 Motion Blur 和 Depth of Field
限制只能减少不能消除走样（仍是点采样）需要更高位深防溢出
Q3. Stochastic Sampling 相比 Supersampling 在视觉质量上有什么优势？原理是什么？

优势：相同样本数下，Stochastic Sampling 比 Supersampling 产生更可接受的视觉效果。

原理：Supersampling 使用规则网格采样 → 走样是有规律的锯齿伪影（人眼对其非常敏感）。Stochastic Sampling 使用非均匀随机采样点 → 走样被转化为随机噪声（人眼对随机噪声的容忍度远高于规律锯齿）。这是利用人眼视觉特性优化感知质量的策略。

Q4. Catmull 算法为什么被称为"最精确"的反走样方法？它的主要缺点是什么？

精确的原因：Catmull 算法将每个多边形精确裁剪到每个像素区域，计算实际覆盖面积（而非点采样），然后做面积加权颜色平均。所有其他方法都是对覆盖率的近似（点采样或子采样），而 Catmull 是精确计算。

主要缺点：多边形裁剪和深度排序的计算量极大，在实际应用中效率过低，不适合实时渲染。A-buffer 是对 Catmull 思路的实用化改进。

Q5. A-buffer 方法中"子像素掩码（subpixel mask）"的作用是什么？为何 A-buffer 的代价只与可见碎片数有关？

子像素掩码的作用：用一个 bitmask（如 4×4 = 16 bits）表示一个多边形碎片覆盖了像素内哪些子像素位置。通过位运算（bitwise AND/OR）可以高效计算各碎片的覆盖率，无需精确的几何裁剪。

代价与可见碎片数有关：A-buffer 每个像素维护一个链表，只存储覆盖该像素的碎片。与 Supersampling 不同（需要对所有 k² 个子像素做全分辨率渲染），A-buffer 的计算量取决于每个像素被多少个多边形覆盖。简单场景（大多像素只被 1-2 个面覆盖）下代价很低。

Q6. (综合) 为什么说"超采样不能消除走样"？哪种方法在理论上可以消除走样？

超采样不能消除走样：因为超采样本质上仍然是点采样——只不过用更密的 k×k 个子采样点代替了原来的 1 个采样点。只要采样密度是有限的（k 是有限的），就仍然存在未被采样的高频信号，走样就不可能完全消失。超采样只能让走样在视觉上减少到可以接受的程度。

理论上可消除走样的方法：

  • Catmull 算法——不依赖点采样，而是精确计算多边形在像素内的覆盖面积（面积加权），因此理论上没有走样。
  • Stochastic Sampling——当样本数趋于无穷时，随机采样的期望值收敛到真实值（但有限样本时仍有噪声）。
Q7. (真题变形) 解释时间走样的两种产生原因，并各举一个例子。

原因 1：空间走样引起（主要原因）

移动物体在连续帧之间，空间走样的表现不同 → 相邻帧图像突变。例如：一个比像素略大的三角形在水平移动时，不同帧落在不同采样点位置，导致锯齿形状持续"跳动"。

原因 2：时间域欠采样

采样频率（帧率）低于运动频率的 2 倍。例如：车轮反转效应（Wagon Wheel Effect）——车轮高速旋转，但低帧率摄像机采样到的帧恰好使轮辐看起来在逆向旋转。

Q8. 给定场景时如何快速选择合适的反走样方法？

边缘锯齿明显、实现简单：Supersampling。它最容易解释，但内存和像素量按 k² 增长。

内存受限但可多次渲染：Accumulation Buffer。它只需正常分辨率缓冲，代价是多次完整渲染，并能顺带实现 Motion Blur / Depth of Field。

规则锯齿很刺眼：Stochastic Sampling。把规律伪影变为随机噪声，视觉上更可接受。

题目强调理论精确覆盖率：Catmull's Algorithm。按面积加权，最精确但计算量最大。

题目出现透明、多片段、subpixel mask：A-buffer。它是 Catmull 思想的实用化近似。

最后速记

必背定义 / 概念

  • 走样根本原因：点采样（Point Sampling）
  • Nyquist：N ≥ 2Fmax
  • 空间走样三表现：Jaggies / 细小物体消失 / 形状扭曲
  • 时间走样主因：空间走样导致
  • 时间走样次因：时间域欠采样 → Wagon Wheel Effect
  • Supersampling：高分辨率渲染 → 降采样。只能减少不能消除
  • Accumulation Buffer：多次渲染 → 累加。额外支持 Motion Blur + DOF
  • Stochastic：随机采样 → 锯齿变噪声
  • Catmull：精确面积加权 → 理论最优
  • A-buffer：链表 + bitmask → 代价与片段数相关

考场容易遗漏的细节

  • 超采样"不能消除"走样——考场上极易写成"消除"
  • Accumulation Buffer 的 Motion Blur / DOF 是相对普通 supersampling 的额外优势
  • 随机采样的"随机"是指采样点位置随机，不是颜色随机
  • Catmull 和 A-buffer 的关系：Catmull 是理论框架，A-buffer 是其实用化
  • A-buffer 的子像素掩码本质仍是超采样（固定子像素位置）
  • Stochastic Sampling 的三个分布记得住：Poisson-Disc（最常用）、Jittered、完全随机
  • 时间走样的两类原因要分开答，不要说"时间走样就是空间走样"
  • Nyquist 定理映射：采样频率 = 分辨率，信号频率 = 图像高频细节
